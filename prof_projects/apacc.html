Asian Pacific American Community Center

Community Data & Outreach Analytics

Overview

This project focused on transforming APACC's manual, paper-based workflows into a data-driven system supported by relational database design and SQL-based analysis. The methodology emphasized structured data modeling, query-driven insights, and process optimization — translating raw data into process improvements and strategic recommendations. By integrating SQL with process analysis, the project moved APACC from reactive, manual workflows to a more scalable, data-informed operational model.

1. Data Structuring & Database Design

A relational database schema was designed to centralize operational data across key entities including Clients, Applications, Appointments, and Staff — eliminating fragmented record-keeping and enabling efficient querying.

  • Defined primary and foreign keys to establish relationships between entities
  • Normalized tables to reduce redundancy and improve data integrity
  • Structured fields such as client_id, days_per_week, age, and application_status to support analytical queries

2. Query Development & Exploratory Data Analysis

SQL was used as the primary tool to explore patterns in daycare application and appointment data.

Age Distribution Analysis

SELECT age, COUNT(age) AS client_frequency
FROM daycare_application
GROUP BY age
ORDER BY age ASC;

Identified a concentration of applicants around ages 7–10, with a mean age of approximately 9 years old.

Acceptance Rate Calculation

SELECT accepted_or_denied AS application_status,
       COUNT(*) AS applicant_count,
       ROUND((COUNT(*) * 100.0 / SUM(COUNT(*)) OVER()), 2) AS acceptance_rate
FROM daycare_application
GROUP BY accepted_or_denied;

Revealed a 70% acceptance rate and 30% denial rate, highlighting opportunities to improve applicant support.

Attendance Behavior Segmentation

SELECT visits_per_week, COUNT(*) AS quantity
FROM (
  SELECT days_per_week,
         CASE
           WHEN days_per_week <= 2 THEN 'Low'
           WHEN days_per_week BETWEEN 3 AND 4 THEN 'Medium'
           WHEN days_per_week >= 5 THEN 'High'
         END AS visits_per_week
  FROM daycare_application
) AS categorized_visits
GROUP BY visits_per_week
ORDER BY quantity DESC;

Showed that most clients fall into medium (3–4 days) and high (5+ days) usage categories, indicating consistent engagement and demand for services.

Join-Based Behavioral Insights

SELECT visits_per_day, reasoning, COUNT(*) AS number_of_clients
FROM (
  SELECT c.client_id,
         CASE
           WHEN days_per_week <= 2 THEN 'Low'
           WHEN days_per_week BETWEEN 3 AND 4 THEN 'Medium'
           WHEN days_per_week >= 5 THEN 'High'
         END AS visits_per_day
  FROM daycare_application c
) c
JOIN appointment a ON c.client_id = a.client_id
GROUP BY visits_per_day, reasoning
ORDER BY visits_per_day, number_of_clients DESC;

Connected visit frequency with service needs (e.g., English, Math, Science), enabling more targeted program planning.

3. Process Optimization Through Data

Insights derived from SQL directly informed process redesign decisions:

  • High visit frequency → required better scheduling and staff allocation
  • Age clustering → guided resource planning and staffing specialization
  • Application drop-offs → highlighted need for validation and clearer instructions

These findings justified the implementation of a database-backed appointment scheduling system and a digital application form with built-in validation — reducing manual inefficiencies and improving data accuracy.

4. Impact & Technical Takeaways

  • Designed normalized relational databases for operational systems
  • Wrote analytical SQL queries to extract actionable insights
  • Applied aggregation, window functions, CASE statements, and joins to model real-world behavior
  • Translated raw data into process improvements and strategic recommendations
