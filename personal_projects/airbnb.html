Airbnb Demand Prediction Model

Machine Learning & Predictive Analytics Project

Project Overview

This project focused on identifying the key factors that distinguish high-demand from low-demand Airbnb listings across major California cities. The goal was to help hosts and investors better understand what drives booking activity and optimize listing performance.

  • Analyzed pricing, location, and listing characteristics
  • Built models to classify high-demand vs. low-demand listings
  • Generated insights for hosts, managers, and investors

Data Sources & Scope

We worked with large-scale public Airbnb datasets across multiple markets to ensure robust analysis.

Combined data from:

  • Los Angeles (~45K listings)
  • San Francisco (~36K listings)
  • San Diego (~13K listings)
  • Final dataset included ~95,000 listings and 19 features
  • Key variables: price, room type, location, reviews, availability, and minimum nights

Data Cleaning & Feature Engineering

Significant preprocessing and feature engineering were required to prepare the dataset for modeling.

  • Cleaned price data by removing currency symbols and formatting
  • Handled missing values: numerical → filled with median; categorical → filled with mode or "Unknown"
  • Created a custom Demand Score to avoid data leakage: 70% review frequency + 30% review recency
  • Classified listings into High-Demand (top 30%) and Low-Demand (bottom 30%)

Model Development & Evaluation

We built and compared multiple models to predict listing demand and evaluate performance.

  • Logistic Regression (baseline): Accuracy 68.89%, AUC 72.41%
  • Random Forest (final model): Accuracy 93.28%, AUC 97.90%
  • Random Forest significantly outperformed baseline models in both accuracy and classification performance

Key Drivers of Demand

The analysis identified the most important factors influencing Airbnb listing performance.

  • Number of reviews — strongest indicator of trust and popularity
  • Minimum nights & availability — shorter stays and higher availability increase demand
  • Price & location — competitive pricing and desirable neighborhoods drive bookings

Insights & Model Performance

The Random Forest model demonstrated strong predictive capability in distinguishing listing demand levels.

  • High AUC score indicates excellent classification performance
  • Model performs well across both high-demand and low-demand listings
  • Confirms that demand can be effectively predicted using behavioral and listing features

Recommendations

Based on the analysis, we developed actionable strategies for both hosts and customers.

For Hosts:

  • Encourage reviews to build social proof
  • Maintain flexible minimum night requirements
  • Use competitive and dynamic pricing strategies
  • Highlight location advantages (walkability, attractions, etc.)

For Customers & Investors:

  • Prioritize listings with strong review activity
  • Look for flexibility in stay duration
  • Evaluate price-to-value balance when selecting properties

Key Insights

This project demonstrates how machine learning can be used to uncover actionable insights in real-world marketplaces.

  • Demand is driven by a combination of trust signals, pricing strategy, and availability
  • Ensemble models like Random Forest provide high predictive accuracy for classification tasks
  • Data-driven insights can support better decision-making for both hosts and platform stakeholders
